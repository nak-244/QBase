# ■ QBase 障害対策（自動復旧・定期再起動）設定メモ

---

## ■ 背景

* 毎週日曜夕方にCloudflare経由で接続不可となる事象が発生
* 同時にローカル（localhost:4000）も応答不可になるケースあり
* Dockerコンテナ自体は稼働状態のまま（停止ではなく応答不能）

---

## ■ 原因の整理（現時点）

* cloudflared：正常稼働
* Docker：稼働状態維持
* OOM（メモリ不足）：発生なし
* → 一時的なアプリ応答不能（OpenWebUI / Ollama / WSLのいずれか）が原因と推定

---

## ■ 対応方針

* 完全停止ではなく「一時的な応答不能」に対する対策
* 方針：

  * 自動復旧（検知 → 再起動）
  * 定期リフレッシュ（予防再起動）

---

## ■ 実施内容

### ① cronによる自動復旧（深夜のみ）

```bash
*/5 0-6 * * * curl -f --max-time 5 http://localhost:4000 || docker restart open-webui
```

#### 内容

* 0:00〜6:59の間、5分ごとに実行
* localhost:4000へ疎通確認
* 応答不可の場合のみコンテナ再起動

---

### ② 週次予防再起動

```bash
0 12 * * 0 docker restart open-webui
```

#### 内容

* 毎週日曜 12:00 に強制再起動
* 長時間稼働による不安定化をリセット

---

## ■ cron稼働確認

```bash

```
