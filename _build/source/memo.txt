了解しました。余計な分岐を排除して、**このPCでの開発環境（QBase-Dev）の標準手順**だけを一本化します。
目的は「毎回迷わず同じ手順でビルド→起動できる状態」です。

---

# ■ 前提（ここだけ守る）

* Ollamaは **systemdで常時起動（0.0.0.0）**
* ソースは **Dドライブ**
* Dockerは **WSL上で実行**

---

# ■ 全体フロー（これだけ覚えればOK）

① ソースに移動
② ビルド
③ コンテナ起動
④ ブラウザ確認

---

# ■ 手順①：ソースに移動

```bash
cd /mnt/d/QBase/_build/source/open-webui-source-v0.8.9-dev
```

---

# ■ 手順②：ビルド

通常：

```bash
docker buildx build --load -t qbase-dev:v0.8.9.1 .
```

不具合時のみ：

```bash
docker buildx build --load --no-cache -t qbase-dev:v0.8.9.1 .
```

---

# ■ 手順③：コンテナ起動

```bash
docker rm -f open-webui-dev
```

```bash
docker run -d \
  --restart unless-stopped \
  -p 4000:8080 \
  -e WEBUI_NAME="QBase-Dev" \
  -e OLLAMA_BASE_URL=http://host.docker.internal:11434 \
  -e WEBUI_DISABLE_VERSION_CHECK=true \
  -e ENABLE_OLLAMA_EMBEDDINGS=true \
  -e EMBEDDING_ENGINE=ollama \
  -e EMBEDDING_MODEL=mxbai-embed-large \
  -e USE_EMBEDDING_MODEL_DOCKER="" \
  -e RAG_EMBEDDING_MODEL="" \
  -e TZ=Asia/Tokyo \
  --add-host=host.docker.internal:host-gateway \
  -v open-webui-dev:/app/backend/data \
  --name open-webui-dev \
  qbase-dev:v0.8.9.1
```

---

# ■ 手順④：確認

ブラウザ👇

👉 [http://localhost:4000](http://localhost:4000)

---
